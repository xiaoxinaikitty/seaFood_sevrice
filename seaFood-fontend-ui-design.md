# 基于 Spring Cloud 的海产品服务平台前端界面设计文档

## 1. 文档说明

### 1.1 文档目的
本文档基于 [demand.md](E:/巢湖学院毕设系统/基于Spring%20Cloud的海产品服务平台/seaFood_frontend_user/demand.md) 的业务需求，输出海产品服务平台的前端界面设计方案，用于指导 React 前端页面设计、组件拆分、交互实现和视觉统一。

### 1.2 适用范围
- 用户端界面设计
- 供应者中心界面设计
- 管理端界面设计
- 公共视觉规范与组件规范

### 1.3 设计原则
- 以海洋、生鲜、清爽、可信赖为核心视觉气质。
- 以“信息清晰、路径直达、操作明确”为核心交互目标。
- 页面设计兼顾演示效果与工程可实现性，避免过度复杂的炫技式交互。
- 用户端强调内容浏览、商品展示和供需流转；管理端强调效率、可读性和数据监管。

## 2. 整体设计目标

### 2.1 用户端目标
- 让游客快速理解平台定位与主要内容。
- 让普通用户便捷完成浏览、收藏、发布求购、提交模拟订单等操作。
- 让供应者能在较短路径内完成商品发布、供应信息管理和订单处理。

### 2.2 管理端目标
- 让管理员快速掌握平台整体运行状态。
- 让审核、管理、统计类操作具备较高效率。
- 通过统一布局和表格化界面体现后台系统的规范性与专业性。

### 2.3 设计落地目标
- 页面布局标准化，便于 React 组件复用。
- 视觉变量可配置，便于统一主题与后期调整。
- 兼顾桌面端与移动端浏览体验。

## 3. 设计风格定义

### 3.1 视觉关键词
- 海洋感
- 新鲜感
- 清洁感
- 秩序感
- 学习型平台感

### 3.2 风格方向
用户端采用“海洋蓝 + 冰川白 + 海产橙”的现代内容平台风格，整体视觉偏明亮、通透、层次清晰。页面背景不使用纯白平铺，而是通过浅海蓝渐变、柔和光斑、波纹线条或几何切面营造海产品平台氛围。

管理端采用更克制的浅色工作台风格，以高信息密度和清晰层级为主，避免使用过多装饰性图形。

### 3.3 风格基调区分
- 用户端：更强调品牌感、图片感、推荐感和内容吸引力。
- 管理端：更强调结构化、数据感、操作效率和状态识别。

## 5. 字体与排版设计

### 5.1 字体建议
- 中文字体：`"Noto Sans SC", "Microsoft YaHei", sans-serif`
- 英文字体：`"Barlow", "Noto Sans SC", sans-serif`
- 数据字体：`"JetBrains Mono", "Consolas", monospace`

### 5.2 字号层级
- 一级标题：32px / 700
- 二级标题：24px / 700
- 三级标题：20px / 600
- 卡片标题：18px / 600
- 正文：14px / 400
- 辅助说明：12px / 400
- 数据大字：36px / 700

### 5.3 排版原则
- 采用大标题 + 简短描述 + 功能内容的层级组织方式。
- 列表页以“筛选区 + 内容区 + 分页区”为统一结构。
- 详情页采用“主信息区 + 辅助信息区 + 扩展内容区”布局。
- 文字段落控制在便于扫描的长度，避免大段密集文本。

## 6. 栅格与布局规范

### 6.1 用户端布局
- 页面主体内容宽度建议为 `1200px`。
- 超宽屏时主体居中，左右保留留白。
- 首页可使用局部满宽区域增强视觉冲击，但内容容器仍建议固定宽度。

### 6.2 管理端布局
- 采用经典后台布局：左侧导航 + 顶部栏 + 内容工作区。
- 侧边栏宽度建议 `220px`。
- 内容区采用卡片化布局，页面内边距建议 `24px`。

### 6.3 栅格规则
- 用户端推荐 12 栏栅格。
- 卡片间距建议使用 16px 或 24px。
- 模块间主间距统一为 32px。

### 6.4 圆角与阴影
- 小型组件圆角：8px
- 卡片圆角：16px
- 弹窗圆角：20px
- 卡片阴影：`0 10px 30px rgba(14, 116, 144, 0.08)`
- 悬浮态阴影：`0 16px 40px rgba(15, 23, 42, 0.12)`

## 8. 用户端界面设计

### 8.1 首页设计

#### 8.1.1 页面目标
- 快速建立平台认知。
- 强调平台主题、热门海产品和供需活跃度。
- 引导用户浏览商品、查看资讯和发布信息。

#### 8.1.2 页面结构
1. 顶部导航栏
2. 首屏品牌展示区
3. 热门海产品推荐区
4. 最新供应信息区
5. 最新求购信息区
6. 海产品知识文章区
7. 平台公告区
8. 页脚

#### 8.1.3 首屏设计建议
- 使用大幅海洋渐变背景，叠加海鲜食材或市场场景图片。
- 左侧展示平台标题、平台简介、主行动按钮。
- 右侧展示海产品卡片拼贴或海鲜主视觉图。
- 增加数据标签，如“商品分类数”“供应信息数”“活跃用户数”。

#### 8.1.4 推荐区设计建议
- 热门海产品采用 4 列卡片布局。
- 供应和求购区采用左右分栏列表布局。
- 资讯区采用 1 张主图卡片 + 3 条列表的混合结构。

### 8.2 登录与注册页设计

#### 8.2.1 设计目标
- 简洁、可信赖，减少干扰。

#### 8.2.2 布局建议
- 左侧品牌视觉区，右侧表单区。
- 表单卡片置于浅色背景中央。

#### 8.2.3 字段设计

- 密码
- 记住登录

注册：

- 手机号
- 密码
- 确认密码
- 验证码

## 9. 供应者中心界面设计

### 9.1 设计原则
- 与用户端保持同一视觉语言，但提升操作效率。
- 对供应者高频操作进行集中展示。

### 9.2 概览页
- 展示商品总数、在售商品数、供应信息数、待处理订单数、浏览量等数据卡片。
- 下方展示最近订单、最近审核结果、快捷操作入口。

### 9.3 我的商品页
- 顶部提供“发布商品”按钮。
- 列表采用表格 + 状态标签 + 快捷操作。
- 支持按审核状态、上架状态筛选。

### 9.5 订单处理页
- 列表建议使用表格。
- 每行提供确认订单、标记发货、查看详情等操作。

## 12. 交互设计规范

### 12.1 基础交互规则
- 所有点击元素必须有悬浮态、按下态和禁用态。
- 主按钮颜色突出，次按钮保持克制。
- 危险操作必须二次确认。

### 12.2 列表交互规则
- 支持分页。
- 支持条件筛选。
- 空结果时给出清晰引导。
- 删除类操作完成后给出反馈提示。

### 12.3 表单交互规则
- 实时校验格式性错误。
- 提交失败时明确指出错误字段和原因。
- 长表单支持自动保存草稿可作为增强功能。

### 12.4 状态反馈规则
- 加载中显示骨架屏或局部 loading。
- 成功状态使用绿色反馈。
- 异常状态使用红色提示并给出下一步建议。

### 12.5 动效建议
- 首屏模块加载可采用淡入和轻微上移动画。
- 卡片 hover 采用轻微抬升与阴影增强。
- 页签切换和弹窗出现采用 150ms 到 250ms 的平滑过渡。

## 13. 响应式设计规范

### 13.1 断点建议
- 大屏桌面：`>= 1440px`
- 标准桌面：`1200px - 1439px`
- 平板：`768px - 1199px`
- 手机：`< 768px`

### 13.2 用户端响应式策略
- 首页推荐区在移动端改为单列堆叠。
- 顶部导航在移动端折叠为抽屉菜单。
- 商品卡片由四列降为两列或单列。
- 筛选条件在移动端收纳为筛选抽屉。

### 13.3 管理端响应式策略
- 优先保证桌面端完整体验。
- 平板端可折叠左侧导航。
- 手机端不建议完整承载后台复杂操作，仅保障基本查看能力。

## 14. 页面状态设计

### 14.1 空状态
- 商品为空：展示“暂无海产品，稍后再来看看”
- 订单为空：展示“你还没有提交任何订单”
- 消息为空：展示“当前没有新消息”

### 14.2 错误状态
- 网络错误：提供重试按钮
- 权限不足：展示权限说明和返回入口
- 数据不存在：展示返回列表按钮

### 14.3 加载状态
- 首页模块使用骨架卡片
- 列表页使用表格骨架或卡片骨架
- 详情页主图和文本区域分别加载

## 17. 最终界面设计结论

本系统前端界面应采用“用户端偏内容与展示、管理端偏结构与效率”的双体系设计方案：

- 用户端以海洋清新风格为主，突出海产品展示、供需信息、资讯内容与个人中心。
- 供应者能力不单独拆成独立系统，而是并入个人中心形成供应者工作台。
- 管理端采用标准后台工作台风格，以审核、管理、统计为核心。
- 全站通过统一颜色变量、卡片规范、状态标签和布局规则保证视觉一致性。

该设计方案既符合学习型海产品服务平台的项目定位，也具备较强的前端落地可实现性，可直接作为后续 React 页面开发和 UI 细化的基础文档。
